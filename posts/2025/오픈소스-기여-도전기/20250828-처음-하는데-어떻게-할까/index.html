계기#

오랜만에 깃헙 프로필을 보았다. 내 소개문구 쪽에 지우긴 했지만, 내가 이전에 올려놓은 소개 문구에서 ‘오픈소스 기여에 관심이 있다’고 말해두었었다.

빈말이 아니라 정말로 오픈소스 쪽에 기여를 해보면서 커뮤니티 경험을 쌓고 싶었고, 한번 이제 도전해보려고 한다. 백수(?)가 되기 전에 오픈소스에서라도 취업(?)을 해야하지 않을까?

처음은 아니다#

사실 지인의 추천으로 간단하게 리액트의 전화번호 컴포넌트 라이브러리의 한국어 번역을 조금 도와주긴 했다.

https://github.com/jackocnr/intl-tel-input

실제로 어색한 표현이 있었고, 정말 좋은 기회라고 생각하여 발 빠르게 PR을 보냈다 ㅎㅎ.

어센션 섬이라는 영국령의 행정구역(?) 이란 곳도 있는데, 이 부분이 그냥 그대로 승천섬이라고 번역되어 있었다.

https://ko.wikipedia.org/wiki/%EC%96%B4%EC%84%BC%EC%85%98%EC%84%AC

여하튼 우리나라 표기상 어센션 섬이라고 되어있으니 이렇게 바꿔주며, 검색 결과에 관한 번역 기록도 내가 고쳐주었다.

그래서 아마 실제로 내가 고친 결과를 쓰는 사람도 실제 사람들 중에도 있을거다..? 전화번호 컴포넌트를 쓰는 사람들은 저 컴포넌트를 쓸 테니 말이다.

그런 생각을 하니까 진짜 회사의 일원(?)이 된 거 같아서 기분이 좋았었다. 이런 느낌을 다시 느껴보고 싶다.

최종 목표#

일단 최종 목표는 FFmpeg 에 기여해보는게 목표.

목표가 너무(?) 높은거 아닌가 싶기도 할 텐데.. 결국 최종 목표니까 천천히 해보려고 한다. 몇년이 걸릴지도 모르겠으나 메이플같은 게임도 정점은 있지만 맹목적으로 쫓지는 않는 그런 느낌? 으로 해보려고 한다.

게다가 워낙에 오픈소스에는 또라이(?)들이 많기 때문에.. 솔직히 걱정도 많기도 하기에 일단 당장은 쉬운 오픈소스를 도전해보지 않을까 싶다.

지금 당장은?#

그래서 지금 당장은 일단 초기 프로젝트에 참여해보는걸 목표로 하고자 한다.

안그래도 주변에서도 다들 쉬운 프로젝트부터 시작하라고 했는데, 마침 AI가 검색하는 방법을 알려주었다.

내가 아예 모르지만 쉽게 널리 존재하던 무언가를 찾아주는건 AI가 편한 것 같다. 원래라면 검색을 몇날 며칠을 하는 건데 이건 정말 좋은듯 하다.

오픈소스 찾는 방법#

오픈소스 찾는 방법을 간단히 알려주겠다.

일단은 깃헙에서 이렇게 오픈소스 커밋을 시작하기 위한 사람들에게 오픈마인드인(?) 분들이 친절하게 태그를 달아두신다고 한다.

대략적으로 아래처럼 말이다.

  • good first issue, help wanted, beginner, easy 등의 라벨은 신규 기여자를 환영하는 이슈를 찾기 위한 가장 좋은 방법이다.

  • GitHub 검색창에 video processing good first issue 또는 webrtc help wanted 와 같이 관심 분야 키워드와 위 라벨들을 조합하여 검색해보자.

그리고 친구들 중에 한명이 마이너한 언어 (마크 커맨드) 를 사용하면서 직접 커뮤니티에 들어가서 질문글들을 많이 눈팅하면서 도움을 얻었다고 한 기억이 났었다.

그렇기 때문에, 일단은 관심 있는 특정 기술이나 소프트웨어의 공식 웹사이트나 커뮤니티(포럼, 디스코드, 메일링 리스트 등)에 참여해 보자.

어떤 프로젝트들이 활발하게 개발되고 있는지, 어떤 도움이 필요한지에 대한 정보를 직접 얻을 수 있는 좋은 방법이다.

일단은 깃허브 검색창에 직접 아래처럼 입력해보았다.

label:"good first issue" is:issue is:open

이렇게 깃허브에서는 검색 태그를 이용해서 필터링을 조금 더 세심하게 할 수 있는데, 이렇게 좋은 기능을 잘 사용하지 못하고 있었던 것 같다.

이제 여기서 조건을 더 추가해보는 것이다.

나같은 경우는 언어를 Java 로 해보고 싶었기 때문에, 이렇게 추가해보았다.

label:"good first issue" is:issue is:open language:Java

이렇게 추가하고, 이제 스타 개수도 적당하면서 기여 환경도 적당한(회사 생활처럼 이것도 눈치를 잘 봐야하지 않을까..? 가령 신입 기여자도 잘 받아주는 그런 곳인지 확인해볼 필요가 있다..) 곳을 골라보았다.

나는 여기서 processing 이 좋아보여서 한번 포크를 해 보았고, 이제 해당 프로젝트가 어떤 프로젝트인지 직접 돌려보면서 실행해보고, 공부해보면서 프로젝트에 익숙해졌을때 기여를 해보면 된다!
